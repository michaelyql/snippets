# CF1909D - Split Plus K

## Summary

- Given $a_1,\dots,a_n$ initially written on a board and an integer $k$, in one move choose any $x$ on the board and replace it with two positive integers $y$ and $z$ such that $y+z=x+k$.
- Find the minimum number of moves to make all elements the same, or report that it is impossible.
- Constraints: 
    - $1\leq n\leq 2\cdot 10^5$
    - $1\leq a_i, k\leq 10^12$

[Link to Problem Statement](https://codeforces.com/problemset/problem/1909/D)

## Solution

- Consider how the array must look like during the last move:
    - It must look like $[x,\dots,x,2x-k]$
    - In the second last step, the array must either look like $[x,\dots,x,2x-k,2x-k]$ or $[x,\dots,x,3x-2k]$
    - In the third last step, the array must be either $[x,\dots,x,2x-k,2x-k,2x-k]$ or $[x,\dots,x,2x-k,3x-2k]$ or $[x,\dots,x,4x-3k]$
- In general, any number in the array must be of the form $px-(p-1) k$ for $p\geq 1$
    - So for $a$ to be convertible to $[x,..,x]$ all $a_i$ must be of the form $px-(p-1) k$
    - $\Rightarrow a_i + (p-1) k = px$
    - $\Rightarrow a_i - k + kp = px$
    - $\Rightarrow a_i - k = p(x-k)$
- We need $x$ to be a positive integer, since in each op we can only write two positive integers $y$ and $z$ on the board, so values always stay positive at all times
- Casework:
    - If $x<k$, then $x-k$ is negative. Since we must have $p >= 1$, this must mean all $a_i-k$ must be negative
    - Else if $x > k$, then $x-k$ is positive. Since $p >= 1$, it must mean all $a_i-k$ must be positive. 
    - Else if $x == k$, then $x-k$ is 0, then all $a_i-k$ must be equal to 0. 
    - These are the only possible cases. We cannot have both negative and positive $a_i-k$ at the same time. Otherwise there is no valid $x$ and we can report -1
- If these conditions are met, then let $g = \gcd(a_1-k,\dots,a_n-k)$
    - Then $x-k=g$, $x = g+k$ is one possible value of $x$
    - We need $mk + \sum a_i = (n+m)\cdot x$
    - For a fixed $x$, we have $\sum a_i - nx = m(x-k)$ 
    - Then $m = (\sum a_i - nx) / (x-k)$
    - We want to minimize $m$
- Since now we know that $a_i = px - (p-1)k$, and we have several candidates of $x$ ($g$ and its divisors)
    - Each reverse step changes from $x \to 2x-k \to 3x-2k$ and so on
    - So to change from $x$ to $a_i$, we need to take exactly $(p-1)$ steps, and the total number of steps $m = \sum_i (p_i-1) = \sum_i p_i - n $
- We also have $p_i = (a_i-k)/(x-k)$
    - So $\sum_i p_i = \sum_i (a_i-k)/(x-k)$
    - Substituting, $m = \sum p_i - n = \sum_i (a_i-k)/(x-k) - n$
    - Since $\sum (a_i-k)$ is fixed and $n$ is fixed, $m = \sum_i (a_i-k)/(x-k) - n$ is minimized by taking the largest value of $|x-k|$
- If $x>k$, then we want $(x-k)$ as large as possible, so we take $(x-k)=g$
- Else if $x<k$, $(x-k)$ is negative and $\sum(a_i-k)$ is negative
    - So to make $m$ as small as possible, we want to make $(x-k) $as negative as possible, so we take $x-k=-g$ and $x = k-g$
- Overall, a tedious problem to derive the answer having short solution code

## Code

```cpp
ll gcd(ll a, ll b) {
    return b ? gcd(b, a % b) : a;
}

void solve() {
    int n;
    ll k;
    cin >> n >> k;
    vll a(n);
    rep(i,0,n) cin >> a[i];
    int sign = 0;
    if (a[0]-k > 0) sign = 1;
    else if (a[0]-k < 0) sign = -1;

    ll S = a[0]-k;

    rep(i,1,n){
        if (a[i]-k > 0 && sign != 1) {
            cout << -1 << '\n';
            return;
        } else if (a[i]-k < 0 && sign != -1) {
            cout << -1 << '\n';
            return;
        } else if (a[i]-k == 0 && sign != 0) {
            cout << -1 << '\n';
            return;
        }
        S += a[i]-k;
    }
    if (sign == 0) {
        cout << 0 << '\n';
        return;
    }
    ll g = 0;
    rep(i,0,n){
        g = gcd(g, (a[i]-k) * sign);
    }
    if (sign == 1) {
        cout << S / g - n << '\n';
    } else {
        cout << -S / g - n << '\n';
    }
}
```
